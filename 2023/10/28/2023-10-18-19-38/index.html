英语词典

题目描述

编辑距离也称莱文斯坦距离，指的是两个字符串之间，由一个转化为另一个所需的最少编辑操作次数，允许的编辑操作包括：

  1. 将其中一个字符替换为另一个字符
  2. 插入一个字符
  3. 删除一个字符

例如，字符串 "ab" 可以通过修改一个字符变为 "ac"，可以通过增加一个字符变为 "acb"，也可以通过删除一个字符变为 "a"，所以 "ab" 与 "ac", "acb", "a" 的编辑距离都为 111。当然两个相等的字符串的编辑距离为 000。

现在，小 Z 有 nnn 个英语单词，他用这些单词组成了一个英语词典，这样当他遇到一个单词时，就可以通过查词典的方式来帮助记忆。

具体地，小 Z 将 nnn 个单词编辑成词典，然后接下来会遇到 qqq 个需要查询的单词，对于每一个查询单词，如果词典中某一个单词和这个查询的单词的编辑距离 ≤1\le 1≤1，那么就输出词典中的这个单词。如果不存在这样的单词，就输出 No。

「一些出题人善意的提示」

  2. 字符串 sss 与字符串 ttt 的编辑距离 ≤1\le 1≤1，有如下 444 情况：

    • sss 与 ttt 相同；sss 可以通过删除一个字符变为 ttt；sss 可以通过增加一个字符变为 ttt；sss 可以通过修改一个字符变为 ttt。

接下来 qqq 行，每行一个字符串表示小 Z 这次询问的单词。

输出格式

输出共 qqq 行，对于每一次询问输出词典中与询问单词编辑距离 ≤1\le 1≤1 的单词。如果不存在，则输出 No。

小 Z 的游戏币交易

题目描述

小 Z 最近沉迷某款游戏，这款游戏允许玩家在市场中自由交易虚拟游戏币。但是，小 Z 完全没有经商头脑，在交易中每次都是吃亏的那个。

但是，小 Z 是学计算机的。现在他决定重整旗鼓，先自己写一个模拟游戏币交易系统，在这里生成若干个虚拟玩家进行交易，来取得交易市场的一些经验，以便于之后自己在游戏世界里赚的盆满钵。

目前这个系统还比较简单，只有一种叫做「比特币」的虚拟游戏币，「比特币」的交易规则如下：

有 nnn 个玩家参与到游戏币交易过程中，玩家的编号为 1,2,⋯ ,n1,2,\cdots,n1,2,⋯,n，编号 iii 的玩家刚开始共有 aia_iai​ 元启动资金，并且拥有 cic_ici​ 个「比特币」。

现在一共发生了 mmm 次交易，每一次交易共有如下三种不同类型的操作：

  1. 1 id x y

    • 表示编号为 ididid 的人想用 xxx 元每个的单价购买 yyy 个「比特币」。

    • 此时，如果他手上的钱足够 x×yx \times yx×y 元。那么，交易系统会查找售卖池中是否存在卖单，如果有，则会将卖价小于等于 xxx 的卖单，按照卖价从小到大顺序取出，然后依次以卖价进行交易，直到这笔交易完整地完成，或是没有满足条件的卖单为止。如果，最后没有满足条件的卖单，该笔交易还没有被完成，则该笔交易剩余的需求会进入到购买池中进行等待。

    • 如果他手上的钱不足够 x×yx\times yx×y，或者他已经有一笔交易（买单或者卖单）尚未完成，则本次交易被驳回。

    • 上方提到的「售卖池」和「购买池」分别可以理解成一堆卖单或买单组成的交易需求集合。

    • 特别的，当 x=0x=0x=0 时，则买价为当前交易系统最后一次成交的价格，如果没有成交过，则为 000。

  2. 2 id x y

    • 表示编号为 ididid 的人想用 xxx 元每个的单价卖出 yyy 个「比特币」。

    • 此时，如果他手上的「比特币」数量足够 yyy 个。那么，交易系统会查找购买池中是否存在买单，如果有，则会将买价大于等于 xxx 的买单，按照买价从大到小顺序取出，然后依次以买价进行交易，直到这笔交易完整地完成，或是没有满足条件的买单为止。如果，最后没有满足条件的买单，该笔交易还没有被完成，则该笔交易剩余的需求会进入到售卖池中。

    • 如果他手上的「比特币」不足 yyy 个，或者他已经有一笔交易（买单或者卖单）尚未完成，则本次交易被驳回。

    • 特别的，当 x=0x=0x=0 时，则卖价为当前交易系统最后一次成交的价格，如果没有成交过，则为 000。

  3. 3 id

    • 表示编号为 ididid 的人想要撤回自己正在交易的需求。如果他有一笔买单或者卖单尚未完成，则撤回剩余的还没有完成的交易订单。

    • 如果他没有正在交易的需求，则撤回请求被驳回。

现在，小 Z 想知道经过这 mmm 次交易后，一共成交了多少次交易（每一次金额变动算成交一次），以及每个人手上还有多少钱和多少「比特币」？

【注意】

  1. 在交易时，如果有多笔符合条件的订单，则按照价格的优先级排序，如果价格一样，则按照订单生成的时间从小到大排序。这里的价格的优先级取决于是购买池还是售卖池。

  2. 这 mmm 次交易订单是按照时间顺序到达的

【一些出题人的善意提示】

  1. 当 x=0x=0x=0 时，该笔交易订单的买价或者卖家就是当前交易系统最后一笔成交交易的价格，之后就不会再变化了。

  2. 第 iii 次操作产生的交易订单的时间就是 iii，不会随着之后的部分交易的成功而发生变化。

  3. 购买池和售卖池就是由一些还没有成交的订单组成的交易集合。

  4. 请仔细结合样例理解题意。

输入格式

第一行输入两个整数 n,mn,mn,m 分别表示参与交易的玩家数量和交易的操作数。

第二行输入 nnn 个整数，c1,c2,⋯ ,cnc_1,c_2,\cdots,c_nc1​,c2​,⋯,cn​ 表示每位玩家初始拥有的「比特币」数量。

第三行输入 nnn 个整数，a1,a2,⋯ ,ana_1,a_2,\cdots,a_na1​,a2​,⋯,an​ 表示每位玩家初始的启动资金。

接下来有 mmm 行，每行形如：1 id x y, 2 id x y, 3 id 表示一笔交易的操作，操作含义如题。

输出格式

输出共三行。

第一行输出交易成交的次数，每一次交易产生了金额的变动就算一次交易成交（撤回操作不算）。

第二行按照编号顺序输出 nnn 个人剩余的「比特币」数量。

第三行按照编号顺序输出 nnn 个人剩余的金额。

  4. B 想以 666 元每个的单价购买 202020 个「比特币」，售卖池中没有卖单，该交易需求进入购买池。
